## Граница проекта. Контекстная диаграмма

Порядок разработки контекстной диаграммы:  

 - [ ] Из числа заинтересованных лиц собирается рабочая группа (обычно
       от 3 до 5 человек)
 - [ ] Рабочая группа фиксирует в центре диаграммы  **название** 
       конкретной системы
 - [ ] Рабочая группа выдвигает и отображает  **группы**  пользователей,
       которые должны взаимодействовать с системой, обсуждает их
       перечень, дополняет его (группа первого контакта с системой)
 - [ ] Рабочая группа выдвигает и отображает  **смежные системы**,
       которые должны взаимодействовать с системой, обсуждает их
       перечень, дополняет его
 - [ ] Рабочая группа последовательно проходит по каждому элементу
       окружения и описывает  **потоки данных**, связывающие его с
       системой
 - [ ] Рабочая группа проводит  **тестирование**  контекстной диаграммы,
       дополняя диаграмму по ходу тестирования

**Пользователь заходит в систему чтобы _________**
Менеджер склада заходит в систему чтобы управлять каталогом, мониторить остатки, планировать график сборки заказов, просматривать отчеты
Работник склада заходит в системы чтобы провести складские операции, собрать заказ, провести инвентаризацию.
Администратор заходит в систему чтобы управлять пользователями, чтобы управлять доступом пользователей, чтобы настраивать и обновлять систему.
Интернет-магазин заходит в систему чтобы получить информацию о наличии товара
Служба доставки заходит в систему чтобы получить график и место погрузки

Необходимо предоставить **интеграцию** для Интеграции с Интернет-магазином и Службой доставки.
